# AI-Driven Energy Supply Chain Resilience Prototype — Replit Agent Build Brief

## 1. APP OVERVIEW

**App Name:** HormuzWatch — AI Energy Supply Chain Resilience Platform

**One-line description:** A real-time dashboard that monitors geopolitical and shipping risk signals threatening India's crude oil imports, simulates disruption scenarios, and auto-generates ranked procurement rerouting recommendations.

**Core problem it solves:** India imports ~88% of its crude oil, with 40-45% transiting the Strait of Hormuz — a single geopolitical shock (e.g. a Hormuz closure, Red Sea attack, sanctions escalation) can spike prices and strand refiners without alternatives. HormuzWatch compresses the "signal to decision" window from days to minutes by continuously scoring disruption risk, simulating cascading impacts, and surfacing executable alternative sourcing options.

**Target user:** Refinery procurement teams, energy policy analysts, and strategic reserve planners at Indian oil PSUs (IOCL, BPCL, HPCL) or the Ministry of Petroleum.

## 2. FULL FEATURE LIST

### Core (build these first — judged heavily on Innovation + Business Impact)
1. **Live Risk Dashboard** — a corridor-by-corridor (Strait of Hormuz, Red Sea/Bab-el-Mandeb, Persian Gulf) disruption probability score (0-100), color-coded, updating on a timer.
2. **Geopolitical Signal Feed** — an ingested/mocked news + sanctions + shipping-incident feed, tagged by corridor and severity, feeding into the risk score.
3. **Scenario Simulator** — user selects a scenario (Hormuz Partial Closure, OPEC+ Emergency Cut, Red Sea Suspension) and the app computes projected impacts: refinery run-rate drop %, domestic fuel price delta, days-of-SPR-cover remaining, estimated GDP drag.
4. **Adaptive Procurement Recommender** — given a scenario, ranks 3-5 alternative crude sources/routes (e.g. US Gulf Coast, West Africa, Russia ESPO, domestic ONGC) scored on price premium, tanker availability, transit time, and refinery grade compatibility. Output is an actionable table, not prose.
5. **Strategic Reserve (SPR) Optimizer** — shows current SPR days-of-cover (~9.5 days baseline) and a recommended drawdown schedule against the simulated supply gap, with a simple chart.
6. **Geospatial Map View** — interactive map (Leaflet) showing key chokepoints, ports, refineries, and mock vessel positions, color-coded by risk.
7. **Response Timeline / Audit Log** — a log showing "Signal detected → Risk score updated → Scenario triggered → Recommendation generated" with timestamps, to visually prove end-to-end response speed (a judged metric).

### Nice-to-have (build if time remains — polish/wow factor)
8. **AI Chat Assistant** — a chat panel where a user can ask "What happens if Hormuz closes for 10 days?" and get a natural-language summary generated by an LLM call, referencing the simulation outputs.
9. **Alert/Notification banner** — simulated push alert when risk score crosses a threshold.
10. **Scenario comparison view** — side-by-side comparison of 2 scenarios.
11. **Export to PDF** — one-click export of the current recommendation set as a shareable brief (great demo moment).
12. **Dark "command center" mode toggle.**

## 3. TECH STACK

- **Frontend:** React (Vite) + TypeScript + Tailwind CSS + shadcn/ui components
- **Charts:** Recharts
- **Maps:** React-Leaflet + OpenStreetMap tiles (free, no API key)
- **Backend:** Node.js + Express (or Replit's built-in Node server), REST API
- **Database:** Replit's built-in PostgreSQL (via Drizzle ORM) — or in-memory JSON store if time-constrained, seeded with mock data
- **AI/LLM:** Anthropic Claude API (via `@anthropic-ai/sdk`) for: (a) summarizing mock news signals into risk commentary, (b) powering the optional chat assistant, (c) generating natural-language scenario impact narratives
- **Icons:** lucide-react
- **Animations:** Framer Motion (for score changes, card transitions, live-update pulses)
- **Hosting:** Replit's built-in deployment

Keep it a single full-stack repl (client + server folders), no external deployment needed.

## 4. PAGES & USER FLOW

**1. Landing / Command Center (Home — `/`)**
The main dashboard. Shows: top risk score summary bar (3 corridors), live signal feed (scrolling card list), a mini map, and a "Run Scenario" call-to-action button. This is the page judges will spend the most time on — make it visually dense but readable.

**2. Scenario Simulator (`/simulate`)**
User picks a scenario from a dropdown/card selector (Hormuz Closure, OPEC+ Cut, Red Sea Suspension, or Custom % disruption slider). On submit, shows animated loading → results panel with impact charts (refinery run rate, fuel price, SPR days, GDP drag).

**3. Procurement Recommendations (`/recommendations`)**
Populated after a scenario is run (or with default data on first load). Table/card view of ranked alternative sources with scores, a "why this option" expandable detail, and an "Accept Route" mock action button.

**4. Reserve Planner (`/reserves`)**
Shows current SPR levels, a drawdown schedule chart, and a slider to adjust assumed disruption duration, live-updating the projected reserve depletion curve.

**5. Geospatial Map (`/map`)**
Full-screen interactive map with chokepoints, refineries, and vessels as markers; clicking a marker shows a risk detail popup.

**6. AI Assistant (optional, `/assistant` or a persistent side panel)**
Chat interface to ask free-form questions about current risk state.

**Navigation:** Persistent left sidebar (icons + labels) with the 5-6 pages; top bar shows live "Overall Risk Index" badge visible on every page. Flow: Home → click "Run Scenario" → Simulator → auto-redirect/link to Recommendations → Reserve Planner accessible anytime → Map accessible anytime.

## 5. UI & DESIGN INSTRUCTIONS

**Visual identity:** "Energy command center" aesthetic — dark, high-contrast, data-dense, like a NASA mission control or Bloomberg terminal, not a consumer app.

- **Color scheme:** Dark navy/near-black background (`#0B0F1A`), card backgrounds slightly lighter (`#131826`), accent colors: amber/orange (`#F5A623`) for warnings, red (`#EF4444`) for high risk, emerald (`#10B981`) for low risk/safe, electric blue (`#3B82F6`) for primary actions/links. White/light-gray text (`#E5E7EB`).
- **Fonts:** `Inter` for UI text, `JetBrains Mono` or `Space Mono` for numerical data/scores (reinforces the "terminal" feel).
- **Layout:** Sidebar nav + main content grid using CSS grid/flex cards. Generous use of card components with subtle borders (`border-white/10`) and soft shadows. Rounded corners (`rounded-xl`), not sharp, not overly bubbly.
- **Components:** Use shadcn/ui Card, Badge, Tabs, Slider, Dialog, Table components as the base. Risk scores shown as large numeric badges with a colored ring/gauge (use Recharts RadialBarChart).
- **Animations:** Framer Motion for: risk score number count-up/count-down, pulsing dot on "live" signal feed items, smooth panel transitions between scenario states, skeleton loaders during "simulation running" states (fake a 1.5-2s delay for dramatic effect during demo).
- **Polish requirements:** Every page must have zero empty/blank states — always pre-populate with mock data. Add a subtle animated background grid or gradient on the landing page. Use monospace fonts for coordinates, timestamps, and percentages to sell the "intelligence platform" feel.

## 6. DATA & APIS

### Data Models (Drizzle schema / TypeScript interfaces)

```typescript
// RiskSignal
{ id, corridor: "hormuz" | "redsea" | "persian_gulf", source: string, headline: string, severity: "low"|"medium"|"high"|"critical", timestamp: Date, riskDelta: number }

// CorridorRisk
{ corridor: string, currentScore: number (0-100), trend: "up"|"down"|"stable", lastUpdated: Date }

// Scenario
{ id, name: string, description: string, disruptionPercent: number, affectedCorridor: string }

// ScenarioResult
{ scenarioId, refineryRunRateDrop: number, fuelPriceDeltaPercent: number, sprDaysRemaining: number, gdpDragPercent: number, generatedAt: Date }

// ProcurementOption
{ id, sourceName: string, region: string, pricePerBarrel: number, premiumVsBenchmark: number, tankerAvailability: "high"|"medium"|"low", transitDays: number, refineryCompatibilityScore: number (0-100), overallRank: number }

// ReserveStatus
{ currentDaysCover: number, drawdownSchedule: { day: number, projectedDays: number }[] }
```

### APIs to integrate
- **Anthropic Claude API** (`api.anthropic.com`, via SDK) — for generating natural-language risk summaries and powering the chat assistant. Use `claude-sonnet-4-6` (uses server-side key, do not expose in frontend).
- **OpenStreetMap tiles** (free, no key): `https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png` for the Leaflet map base layer.
- No live news/AIS/sanctions API is required for the prototype — **mock these entirely** (see below). If time permits, optionally pull real headlines via a free news API (e.g. NewsAPI free tier) filtered by keywords "Strait of Hormuz," "Red Sea shipping," "Iran sanctions oil" — but this is optional polish, not core.

### Mock/Seed Data (pre-populate on server start)
- 15-20 fake `RiskSignal` entries across the 3 corridors with realistic headlines (e.g. "Houthi forces claim missile strike near Bab-el-Mandeb," "US Treasury sanctions three tankers linked to Iranian crude exports," "Tanker congestion reported at Fujairah anchorage") with varied severities and timestamps over the last 72 hours.
- 3 pre-built `Scenario` presets matching the challenge brief: Hormuz Partial Closure (50% capacity), OPEC+ Emergency Cut (2M bpd), Red Sea Shipping Suspension.
- 5-6 `ProcurementOption` entries per scenario: US Gulf Coast WTI, West African Bonny Light, Russian ESPO Blend, Saudi Arabian Light (reduced allocation), Domestic ONGC crude — with realistic price/premium/transit numbers.
- Baseline `ReserveStatus`: 9.5 days current cover, with a drawdown curve for a 15-day disruption scenario.
- Corridor risk scores baseline: Hormuz 62, Red Sea 71, Persian Gulf 45 (or similar illustrative numbers), refreshing with small random walk every 10-15 seconds on the frontend to simulate "live" updates.

## 7. STEP-BY-STEP BUILD INSTRUCTIONS FOR REPLIT AGENT

1. Initialize a new Replit project using the **Node.js + React (Vite + TypeScript)** template. Set up a `client/` folder for the React app and a `server/` folder for the Express API.

2. Install dependencies: `react-router-dom`, `tailwindcss`, `@radix-ui/*` + `shadcn/ui` CLI setup, `recharts`, `react-leaflet leaflet`, `framer-motion`, `lucide-react`, `@anthropic-ai/sdk`, `express`, `drizzle-orm`, `@neondatabase/serverless` (or use Replit's built-in Postgres binding), `zod` for schema validation.

3. Configure Tailwind with the custom color palette and fonts specified in Section 5 (extend `tailwind.config.js` theme with the navy/amber/red/emerald/blue palette and `Inter`/`JetBrains Mono` font families loaded via Google Fonts in `index.html`).

4. Set up the Express server in `server/index.ts` with REST routes:
   - `GET /api/signals` — returns risk signal feed
   - `GET /api/risk-scores` — returns current corridor risk scores
   - `GET /api/scenarios` — returns list of available scenarios
   - `POST /api/scenarios/:id/run` — runs a scenario simulation, returns a `ScenarioResult`
   - `GET /api/recommendations/:scenarioId` — returns ranked `ProcurementOption[]`
   - `GET /api/reserves` — returns `ReserveStatus`
   - `POST /api/assistant/chat` — takes a user message, calls the Claude API with current dashboard state as context, returns a natural-language response

5. Create the database schema (Drizzle) matching Section 6's data models, or use an in-memory seeded store (`server/mockData.ts`) if you want to skip DB setup for speed — in-memory is acceptable and faster for a hackathon demo.

6. Seed the mock data exactly as described in Section 6 (signals, scenarios, procurement options, reserve baseline) in a `seed.ts` file that runs on server start.

7. Build the client routing in `App.tsx` using `react-router-dom` with routes: `/`, `/simulate`, `/recommendations`, `/reserves`, `/map`. Build a persistent `Sidebar.tsx` component with nav links and a `TopBar.tsx` showing the live overall risk index badge.

8. Build the **Home/Command Center page** (`/`): fetch and display the 3 corridor risk score cards (large numbers + radial gauge via Recharts), a scrollable live signal feed component (map over `/api/signals`, newest first, color-coded by severity, with a pulsing "live" dot animation via Framer Motion), and a mini Leaflet map preview. Add a prominent "Run Scenario Simulation" button linking to `/simulate`.

9. Build the **Scenario Simulator page** (`/simulate`): render the 3 scenario cards as selectable options (plus an optional custom disruption % slider). On selection + submit, call `POST /api/scenarios/:id/run`, show a 1.5-2 second animated "running simulation" loading state (Framer Motion skeleton/spinner), then render the results: 4 stat cards (refinery run-rate drop, fuel price delta, SPR days remaining, GDP drag) plus a bar/line chart via Recharts visualizing the impact.

10. Build the **Recommendations page** (`/recommendations`): fetch `ProcurementOption[]` for the last-run scenario (or default), render as a ranked table/card list sorted by `overallRank`, with expandable rows showing the scoring breakdown, and a mock "Accept Route" button that shows a success toast.

11. Build the **Reserve Planner page** (`/reserves`): show current SPR days-of-cover as a large stat, a slider for assumed disruption duration (1-30 days), and a live-updating Recharts line chart showing projected reserve depletion as the slider moves.

12. Build the **Map page** (`/map`): full-screen React-Leaflet map centered on the Middle East/Indian Ocean region, with markers for Strait of Hormuz, Bab-el-Mandeb, key Indian ports (Mumbai, Kandla, Vizag), and 5-6 mock vessel positions. Color-code chokepoint markers by current risk level. Clicking a marker opens a popup with corridor risk detail.

13. Add the **AI Assistant chat panel** (persistent slide-out drawer, accessible from any page via a floating action button): wire it to `POST /api/assistant/chat`, which on the server side calls the Anthropic API with a system prompt describing the current mock dashboard state (risk scores, active scenario, top recommendation) so responses are grounded and specific.

14. Add a background interval on the frontend (every 10-15 seconds) that fetches `/api/risk-scores` and applies small random fluctuations server-side, so the dashboard visibly feels "live" during a demo — animate the score changes with Framer Motion count-up/down.

15. Add the **Response Timeline/Audit Log** as a component on the Home page or a dedicated panel, showing a sequential log of "Signal Detected → Risk Recalculated → Scenario Available → Recommendation Ready" events with timestamps, to visually demonstrate end-to-end speed.

16. Polish pass: ensure no page has an empty state, add page transition animations, verify dark theme contrast/readability, add a favicon and page title ("HormuzWatch"), and test the full user flow: Home → Run Scenario → view Recommendations → check Reserve Planner → check Map → ask the AI Assistant a question.

17. Add a `PDF export` button (optional, Section 2 item 11) on the Recommendations page using `jsPDF` or `html2canvas` if time remains, generating a one-page summary of the current scenario + top 3 recommendations.

18. Final QA: run through the entire flow once, confirm the Anthropic API key is set as a Replit secret (`ANTHROPIC_API_KEY`), confirm all mock data renders correctly on load with zero manual setup required, and confirm the app is demo-ready with realistic, hackathon-judge-legible numbers throughout.
